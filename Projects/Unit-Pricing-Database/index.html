Unit Pricing Database

A streamlined database solution for construction material pricing estimation

JavaScript TypeScript SharePoint Database Design
Software Contract

Project Background

This project was developed for a construction company that needed an efficient way to store and access unit pricing information for materials used in their projects. Prior to this system, pricing data was scattered across various spreadsheets and documents, making it difficult for estimators to quickly find accurate pricing information when preparing bids. The goal was to create a centralized database with an intuitive interface that would streamline the estimation process.

Solution Design

I designed a comprehensive database structure to organize pricing data by category, supplier, and material type, making it easy to query and retrieve relevant information. The database tracks historical pricing data, allowing the team to identify trends and make more informed decisions about future estimates. Special attention was paid to data validation to ensure pricing information remained accurate and consistent.

The frontend was built on Microsoft SharePoint, leveraging its collaboration features and accessibility within the company's existing infrastructure. I developed custom web parts using JavaScript and TypeScript that provided a clean, user-friendly interface for searching materials, viewing pricing details, and comparing costs across different suppliers. The interface includes filtering options, quick search functionality, and the ability to export data for use in estimation documents.

Key Features

  • Centralized database for all unit pricing information
  • Advanced search and filtering capabilities
  • Historical pricing data tracking and analysis
  • Supplier comparison tools
  • Data export functionality for estimation documents
  • Role-based access control for data management

Technical Challenges and Growth

This project required me to quickly adapt to the SharePoint development environment, which was new to me at the time. I learned how to work within SharePoint's framework while building custom solutions that met the specific needs of the construction company. The experience taught me the importance of understanding client workflows and designing systems that integrate seamlessly into existing business processes.

Working with TypeScript on this project strengthened my understanding of type safety and how it can prevent errors in data-intensive applications. I also gained valuable experience in database design, particularly in structuring data to support efficient queries while maintaining data integrity. The project reinforced the importance of user feedback in iterative development, as several features were refined based on input from the estimators who used the system daily.

Impact

The system significantly reduced the time estimators spent searching for pricing information, allowing them to prepare more accurate bids more quickly. The centralized nature of the database also improved data consistency across the organization and made it easier to update pricing information when suppliers changed their rates. The historical tracking feature proved particularly valuable for identifying seasonal pricing patterns and negotiating better rates with suppliers.
